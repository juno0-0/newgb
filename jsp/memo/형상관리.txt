형상관리
	형태가 없는 것을 관리하는 것

	팀 간 협업을 하기 위한 방법

	1. SVN
		서버에 다이렉트로 간다.

		서버에 있는 데이터를 모든 유저가 직접 공유한다.
		공유하는 절차가 쉽기 때문에 초보자도 쉽게 공유할 수 있다.
		하지만 서버에 문제가 발생하면 복구하기 쉽지 않다(백업파일이 없다면 복구 불가).
			※ 작업을 하나 할 때마다 아카이브 파일로 백업 파일을 반드시 만들자★★★

	2. Git(Git과 Github는 다른 것)
		유저서버(branch)를 거쳐서 서버로 간다.

		서버에 있는 데이터를 유저별 각 서버에서 공유한다.
		공유하는 절차가 까다롭고 복잡하기 때문에 사전지식이 필요하다.
		하지만 서버 문제 발생 시 정상작동되는 한 명의 유저의 서버를 커밋하게 되면
		즉각 복구가 가능하다.

	SVN 서버주소
		riouxsvn.com

		레포지토리 생성 후 팀원 초대(권한은 read&write)

	SVN 이클립스 설정
		help -> Eclipse Marketplace -> Subversive - SVN Team Provider 4.0.5 
		-> install -> 그대로 놔두고 finish -> restart Now
		window -> preferences -> Team -> SVN 클릭 -> 위에 SVN Connector -> Get Connectors...
		-> SVN Kit -> 다음 다음 -> 동의 -> Finish -> restart Now 
		오른쪽 Open Perspective -> SVN Repository Exploring -> open -> 
		왼쪽 프로젝트 창 위에 건전지+ 아이콘 눌러서 url에 https://svn.riouxsvn.com/team4_bjh 붙여넣기
		밑에는 riouxsvn.com에서 가입한 id, pw 입력 -> Finish

	[팀장]
		다시 Open Perspective 누르고 java EE로 돌아가서
		프로젝트 만들고 우클릭 -> Team -> share project... -> 다음다음 -> Commit창
		-> coment에 말 쓰고 finish -> 오른쪽 Open Perspective -> SVN Repository Exploring -> open
		-> 프로젝트창 누르고 새로고침 하면 커밋한 프로젝트가 나온다.

		다시 Open Perspective 누르고 java EE로 돌아가면 로컬에 프로젝트 나온다.
