	수정은 했는데 add는 안했고 
	pull 해야 하는 상황일 경우 사용

git checkout 경로
	특정 폴더 아래의 모든 수정을 되돌릴 때
	혹은 브랜치 이동


commit -> pull -> 수정 -> commit -> push
commit -> pull -> push는 이미 수정을 해서 pull을 못했을 때

pull -> commit -> push 는 작업 시작전

커밋 한번 하면 로컬에 세이브 포인트를 만든 것
푸쉬하면 깃 허브에 세이브 포인트를 만든 것
작업이 끝나면 푸쉬하고 풀리퀘스트는 최종 작업이 끝날 때나 누군가 요청했을 때 해주는 것
