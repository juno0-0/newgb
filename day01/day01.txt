클래스(반)
	타입이다.
	    공통요소(필드)를 매번 새로 선언하지 않고
	    하나의 클래스 안에서 한 번 선언하여
	    가져다 사용한다.
	
객체(인스턴스 변수)
	추상화된 개념을 구체(실체)화 시킨 것.
	클래스의 필드에 접근할 수 있는 변수
		변수라는 개념안에 객체가 들어가 있는 것
		값이 담기면 변수
		주소값이 담기면 객체
	클래스 안에 있는 필드를 인스턴스 변수(객체)라고도 부른다.
		ex) 자동차 클래스
			int 바퀴
			int 창문
			자동차를 구체화 시킨 것

객체화(인스턴스)
	객체를 만드는 작업
	추상적인 개념을 구체(실체)화 시키는 작업
	Car      mom = new    Car(); 
	클래스  객체     Heap   할당된 필드의 주소값

	자동차라는 개념을 설명할 때 구체화 시켜서 설명
		ex) 엄마차의 색깔은 말이야

※ . : 하위 연산자, 멤버변수 접근 연산자
	객체명.필드명
	객체명.메소드명()
	해당 주소로 가라

생성자(Constructor)
	클래스명()

	이름 뒤에 소괄호가 있기 때문에 메소드와 동일한 기능을 가지고 있다.
	하지만 생성자는 리턴이라는 기능을 제공하지 않기 때문에 메소드라고 부르면 안된다.
		할당된 메모리의 주소값을 리턴해야 하기 때문에 다른 것을 리턴하지 않기 위해서
	생성자와 메소드를 구분해야 하기 때문에 메소드는 항상 첫 글자가 소문자, 생성자는 대문자	
	
생성자의 목적
	1. 해당 클래스 필드를 메모리에 할당한 후 할당된 주소값을 리턴한다.
	2. 초기화의 목적
